## 演習の解答
### 演習 1
1. `iterator` は、リストから値を順番に取り出すためのイテレータです。
2. 表示されるのは `10` です。

`next(iterator)` は次の値を 1 つ取り出すので、最初の要素が返ります。

### 演習 2
- `return` は関数を終了して結果を返します。
- `yield` は値をいったん返し、あとで続きから再開できるようにします。

### 演習 3
次のように表示されます。

```python
1
2
```

`yield` が 2 回あるので、`for` 文が順番に 2 つの値を受け取ります。

### 演習 4
`@log_call` は、`greet` 関数の前後に共通処理を追加する役割を持っています。

この例では、呼び出し前に `start` を表示し、呼び出し後に `end` を表示します。

### 演習 5
`with` を使うと、処理の途中で例外が起きてもファイルを閉じやすくなります。

つまり、後片付け漏れを防ぎやすい点が利点です。

### 演習 6
書き換え例は次のとおりです。

```python
from collections import Counter

counts = Counter(["python", "go", "python", "rust", "python"])
```

`Counter` を使うと、出現回数を数える意図がはっきり伝わります。

### 演習 7
1. `yield` を使います。
2. 偶数を 1 つずつ順番に返したいので、ジェネレータが向いています。
3. 例は次のとおりです。

```python
def even_numbers(n):
    for i in range(1, n + 1):
        if i % 2 == 0:
            yield i
```

### 演習 8
考え方の例は次のとおりです。

1. デコレータ関数は元の関数を受け取ります。
2. 内側の関数では時刻を表示してから元の関数を呼び出します。
3. 最後にその内側の関数を返します。

簡単な例は次のようになります。

```python
from datetime import datetime

def show_time(func):
    def wrapper():
        print(datetime.now())
        func()
    return wrapper
```

## クイズの解答
### 問題 1
正解は `1` です。イテレータは値を 1 つずつ順番に取り出す仕組みです。

### 問題 2
正解は `2` です。`yield` は値をいったん返し、そのあと続きから再開できます。

### 問題 3
正解は `1` です。デコレータは関数に共通機能を追加しやすくします。

### 問題 4
正解は `2` です。`@log_call` は、関数をデコレータで包む意味です。

### 問題 5
正解は `2` です。`with` は後片付けを安全に行いやすくします。

### 問題 6
正解は `1` です。`pathlib.Path` はファイルパスを読みやすく扱うときに向いています。

### 問題 7
正解は `1` です。`Counter` は出現回数を数えるのが得意です。

### 問題 8
正解は `1` です。ジェネレータは、必要になった値を順番に処理したい場面で役立ちます。

## 要点
- イテレータは値を順番に取り出す仕組みです。
- ジェネレータは `yield` を使って少しずつ値を返します。
- デコレータは関数に共通処理を追加します。
- `with` は安全な後片付けに役立ちます。
- 標準ライブラリを使うと、読みやすいコードを書きやすくなります。
