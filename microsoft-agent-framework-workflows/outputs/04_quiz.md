# Microsoft Agent Framework Workflows クイズ

## 問題 1
Workflow の主な役割として最も適切なものはどれですか。

1. モデルの学習を行う
2. 複数ステップの流れを制御する
3. Python の文法エラーを自動修正する

## 問題 2
`Executor` の説明として適切なものはどれですか。

1. Workflow 全体の接続ルール
2. 1 ステップの処理を担当する実行単位
3. 環境変数を保存する仕組み

## 問題 3
`Edge` が主に表すものはどれですか。

1. モデルの重み
2. どの Executor からどの Executor へ流れるか
3. Python の import 順

## 問題 4
次のうち、Workflow 向きのケースはどれですか。

1. 単発で短いあいさつ文を返す
2. 1 回だけ数値を足し算する
3. 分類、分岐、承認待ちを含む処理を組む

## 問題 5
`Fan-out` と `Fan-in` の組み合わせが特に役立つ場面はどれですか。

1. 同じ入力を複数観点で並列評価したいとき
2. 変数名を短くしたいとき
3. 画像サイズだけを変えたいとき

## 問題 6
Checkpoint の主な目的として最も近いものはどれですか。

1. 途中状態を保存し、あとで再開しやすくする
2. Python ファイルを圧縮する
3. Edge を自動生成する

## 問題 7
Human-in-the-loop が向いている場面はどれですか。

1. 不足情報を人から受け取りたいとき
2. どんな場合でも必ず使う基本構文として
3. Workflow を使わないとき

## 問題 8
Agent と Workflow の関係として最も適切な説明はどれですか。

1. Agent が流れ全体を表し、Workflow は 1 ステップだけを表す
2. Workflow が流れを管理し、Agent は各ステップの判断に使える
3. 両者は完全に同じ意味で、呼び方が違うだけ

## 問題 9
初学者が Workflow を設計するときの進め方として適切なのはどれですか。

1. 最初から全部入りの並列・分岐・再開機能を実装する
2. まず最小の直列構成を作り、あとから分岐や HITL を加える
3. Executor をなるべく 1 個にまとめる

## 問題 10
次のうち、「状態」の例として適切なのはどれですか。

1. 途中の分類結果
2. キーボードの色
3. エディタのウィンドウ幅
