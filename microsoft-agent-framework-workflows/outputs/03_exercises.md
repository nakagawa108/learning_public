# Microsoft Agent Framework Workflows 演習

## 演習 1: Workflow が必要か判断する
次の処理が `Agent 単体向き` か `Workflow 向き` かを考えてください。

1. 単発でメール件名を 1 行提案する
2. 問い合わせを分類し、担当部署に振り分ける
3. 原稿を 3 観点で並列レビューし、最後に総合コメントを作る
4. 契約申請をチェックし、不足があれば人に確認してから再開する

### ねらい
Workflow を使う場面を見分ける練習です。

## 演習 2: 直列 Workflow を分解する
次の処理を 3 ステップの Executor 名に分けてください。

「会議メモを受け取り、要点を整理し、アクション項目を抜き出し、最後に短い報告文にする」

### 条件
- 1 ステップ 1 役割にする
- できるだけ短い名前をつける

## 演習 3: 分岐を設計する
問い合わせ内容を分類して、次の 3 ルートに分ける Workflow を考えてください。

- 請求
- 技術
- その他

### やること
- 分類 Executor を 1 つ置く
- 分岐先 Executor を 3 つ置く
- どの Edge が条件を持つかを文章で説明する

## 演習 4: 並列レビューを設計する
次の観点で提案書をレビューする Workflow を設計してください。

- 事実確認
- 読みやすさ確認
- リスク確認

### やること
- どこで Fan-out するかを書く
- どこで Fan-in するかを書く
- 最後に何を担当する Executor が必要かを書く

## 演習 5: Human-in-the-loop を入れる
旅費申請 Workflow を考えます。  
申請データに `travel_date` または `amount` が欠けていたら、途中で人に確認してから再開したいです。

### やること
- どの段階で確認を入れるかを書く
- 保存しておくべき状態を 3 つ挙げる
- なぜ checkpoint が必要かを説明する

## 演習 6: 改善前と改善後を比べる
次の「悪い設計」を、Workflow として読みやすい形に改善してください。

悪い設計:
- `process_everything()` という 1 つの Executor が、分類、要約、承認判定、通知まで全部行う

### やること
- 3 から 5 個の Executor に分ける
- 分けた理由を 2 つ書く

## 演習 7: 擬似コードを書く
次の要件を満たす Workflow の擬似コードを書いてください。

- 最初に内容を分類する
- `urgent` なら人の承認へ進む
- `normal` ならそのまま要約して返す
- 承認後は最終回答を返す

### 条件
- `Executor`
- `Edge`
- `checkpoint/resume`

の 3 つの考え方がわかるように書いてください。
