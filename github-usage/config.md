# GitHubの使い方 教材設計

## テーマ名
GitHubの使い方

## 想定読者
- GitとGitHubをこれから学ぶ初学者
- コマンド入力にまだ慣れていない人
- ファイル共有はしたことがあるが、変更履歴の管理は未経験の人

## 学習ゴール
- GitとGitHubの違いを自分の言葉で説明できる
- リポジトリ、clone、commit、push、Pull Request の役割を区別できる
- GitHubの基本画面と、ローカル作業の関係を説明できる
- `clone` → 編集 → `commit` → `push` の流れを短い例で再現できる
- Pull Request が「変更提案の場」であると説明できる

## 教材出力物
- `README.md`
- `outputs/01_overview.md`
- `outputs/02_lesson.md`
- `outputs/03_exercises.md`
- `outputs/04_quiz.md`
- `outputs/05_answers.md`

## 文体
- 初学者向けにやさしく、短い文で説明する
- 例を先に出し、そのあとに意味を説明する
- 専門用語は最初に一言で定義する
- サンプルコードの前後に、何をするコードかを短く添える

## 参照優先順位
1. このテーマ配下の教材ファイル
2. `AGENTS.md` の共通ルール
3. `skills/create-learning-theme/SKILL.md`
4. 初学者向けの一般的なGitHub利用フロー

## 禁止事項
- Gitの内部実装や高度なブランチ戦略に広げすぎない
- コマンドの丸暗記を目的にしない
- GitとGitHubを同じ意味として扱わない
- ローカル作業とGitHub上の作業を混同して説明しない
- サンプルコードだけを置いて、意図や読み方の説明を省かない
