# GitHubの使い方 クイズ

## 問題1
GitHubの説明として最も適切なものはどれですか。

A. Gitの履歴を共有しやすくするサービス  
B. 変更を自動で全部記録するPC内の機能  
C. HTMLだけを保存する専用サイト

## 問題2
`git clone https://github.com/example/study-notes.git` の役割として正しいものはどれですか。

A. 変更を履歴に残す  
B. GitHub上のリポジトリを自分のPCへコピーする  
C. Pull Request を作成する

## 問題3
次の組み合わせとして正しいものはどれですか。

A. `commit`: GitHubへ送る / `push`: ローカルに保存する  
B. `commit`: 変更を記録する / `push`: GitHubへ送る  
C. `commit`: ファイルを削除する / `push`: コメントを書く

## 問題4
`git status` を使う主な目的はどれですか。

A. GitHubのパスワードを変更するため  
B. 今の変更状態を確認するため  
C. Pull Request を承認するため

## 問題5
Pull Request の説明として最も適切なものはどれですか。

A. 変更を取り込んでほしいと提案し、確認してもらう場  
B. ローカルのファイルを自動で圧縮する機能  
C. GitHubアカウントを削除する機能

## 問題6
次のうち、初心者の理解として正しいものはどれですか。

A. PCで保存しただけで、GitHubも必ず更新される  
B. `commit` しただけでは、GitHub上にまだ見えていないことがある  
C. リポジトリには履歴を残せない

## 問題7
次のコードの読み方として正しいものはどれですか。

```bash
git add README.md
git commit -m "READMEを更新"
```

A. 1行目で変更を記録し、2行目でファイル名を変える  
B. 1行目で変更を記録対象にし、2行目で履歴として残す  
C. 1行目でGitHubへ送信し、2行目でPull Requestを作る
